# CIRS Innovation Engine
*Version 2.0*
*Critical Innovation and Research System*

## OVERVIEW

CIRS is a closed, encrypted multi-agent AI innovation system. It runs a
12-stage pipeline that transforms any problem into a fully battle-tested,
multi-perspective innovation blueprint.

The entire source code is encrypted inside core.repack and is never written
to disk in plaintext. Only the runtime memory holds the active code.

## REQUIREMENTS

- Windows 10 / 11 (64-bit)
- PowerShell 5.1 or later (included with Windows)
- Internet connection (for dependency installation and AI API calls)
- A valid API key for one of the supported AI providers:
    Gemini (Google AI Studio)   : aistudio.google.com
    OpenAI (ChatGPT)            : platform.openai.com
    Anthropic (Claude)          : console.anthropic.com
    OpenRouter                  : openrouter.ai


## INSTALLATION
```
Step 1 — Place both files in the same folder:
    setup.ps1
    core.repack

Step 2 — Open PowerShell as Administrator and run:
    powershell -ExecutionPolicy Bypass -File setup.ps1

The installer will:
  [>] Verify core.repack integrity
  [>] Check if Python 3.11+ is installed (installs via winget if missing)
  [>] Install all required Python packages automatically
  [>] Create the workspace and config directories
  [>] Register the global 'cirs' command in your user PATH

Step 3 — Close and reopen your terminal after setup completes.
```

OR Simple Use Powershell to instalations
```
irm https://github.com/Rifanse/CIRS-Ai-Agent-Framework-.git/main/install.ps1 | iex
```

## USAGE

Start CIRS by typing in any terminal:

    cirs

On first launch, CIRS will open the API configuration screen. Enter your
provider and API key to activate the AI agents.

After configuration, type any problem or topic in the input field to start
the full 12-stage innovation pipeline.


## PIPELINE STAGES

The following 12 stages run automatically when you submit a problem:
```
 | Stage  1   Problem Structuring
 | Stage  2   Cross-Domain Research
 | Stage  3   Parallel Solution Exploration  (30-50 diverse hypotheses)
 | Stage  4   Reality Filtering              (Feasibility x Impact x Speed x Cost)
 | Stage  5   Future Simulation              (Short / Mid / Long-term timelines)
 | Stage  6   Multi-Agent Debate             (10 expert personas, 6 debate rounds)
             [USER INPUT REQUIRED HERE]     (Select your preferred strategy direction)
 | Stage  7   Innovation Synthesis           (Cross-idea fusion, emergent design)
 | Stage  8   Failure Injection              (Red-team stress test across 20 dimensions)
 | Stage  9   Iterative Optimization         (Continuous improvement loop)
 | Stage 10   Decision Engine                (Multi-criteria weighted scoring)
 | Stage 11   Knowledge Extraction           (Reusable patterns and principles)
 | Stage 12   Meta Reflection                (AI evaluates its own reasoning biases)
```
User input is only required once at Stage 6 to select a strategic direction.
All other stages run automatically with input locked.


## COMMANDS (type in the CIRS input field)

  <any text>           Run full 12-stage innovation pipeline (DEFAULT)
  /research <topic>    Run deep research only
  /idea <topic>        Run the innovation pipeline explicitly
  /continue            Resume a previously interrupted pipeline session
  /status              Show system status and agent states
  /config              Open API key configuration screen
  /clear               Clear the output log
  /help                Show command reference
  exit or quit         Close CIRS

## OUTPUT LOCATIONS

All files generated by CIRS are placed OUTSIDE the encrypted core.

  Innovation Reports (Markdown)
      %USERPROFILE%\CIRS_Workspace\output\Innovation_<topic>.md

  Research Reports (Markdown)
      %USERPROFILE%\CIRS_Workspace\output\Research_<topic>.md

  Pipeline Session Cache (for /continue)
      %USERPROFILE%\CIRS_Workspace\sessions\_session_cache.json

  API Configuration (API keys and provider settings)
      %USERPROFILE%\.cirs\config.json


## SECURITY

- The source code is AES/XOR encrypted inside core.repack with PBKDF2-SHA256
  key derivation (200,000 iterations).
- The code is decrypted only in memory at runtime.
- No plaintext source files are written to disk during execution.
- Do not distribute core.repack to unauthorized parties.
- The config.json stores your API keys locally on your machine only.
  Never share your config.json file.


## TROUBLESHOOTING

  Issue: 'cirs' is not recognized as a command
  Fix  : Restart your terminal after running setup.ps1

  Issue: Authentication failed error on startup
  Fix  : Re-run setup.ps1 to reinstall core files

  Issue: API call fails or times out
  Fix  : Check your API key in config.json, or use /config to update it
         Ensure your API key has sufficient quota

  Issue: Pipeline stops midway
  Fix  : Type /continue to resume from the last saved checkpoint
